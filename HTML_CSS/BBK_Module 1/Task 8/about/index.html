Reasons for taking the course

As part of my Vocational role I am aiming to make students employable in front end web development through a project based approach. As such I need to upskill to get myself to a proficient standard in Web Development. I have taught computer science for the last 10 years and have some knowledge of web design previously. I am able to use basic CSS on web pages to change appearance but need time to learn how to manipulate layout.

What do I want from the course?

I would like to get a strong understanding of CSS and formatting layout. In the future I want to potentially setup my own Web Apps based around edu tech and would like to start off with HTML / CSS to get a good understanding. Later I want to get learn JavaScript and so might take one a shortcourses in JavaScript.

I have considered whether I should then perhaps look into a framework such as Node.js. I have some experience of PHP from University, although this is 15 years ago now.

I have experience of procedural programming in Python, I have attempted to learn Django before but failed miserably. I then learnt the basics of ruby and attempted to learn rails, again I failed miserably.

Eventually I would like to get to a stage where I can develop to web apps using a suitable language to interogate a database. I have fairly good knowledge of DB design. The big failing I have had whenever I try to get into this is my lack of time, priorities of the day to day job cause issues for me in the long run. I also found that I would get frustrated and try and run before I could walk.

So for this reason I decided to take a step back and begin by getting a detailed understanding of HTML and CSS as starting point

The structure of a course with clear deadline is key to giving me a focus to learn.
